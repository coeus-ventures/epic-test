# Customer Feedback App

A feedback and NPS (Net Promoter Score) application for collecting, analyzing, and acting on customer feedback through surveys.

## Your Task

You are building a web application from scratch. The behaviors in prd.json describe what users should be able to do with the finished app. Your job is to implement all the functionality needed to make those user behaviors work.

## Instructions

Build this application in the CURRENT WORKING DIRECTORY. Do NOT create a subdirectory.

1. **Read progress.txt FIRST** — it shows which behaviors are already implemented and any notes from previous iterations
2. **Read prd.json** — check which behaviors are marked as done (`passes: true`) vs pending
3. **If this is the first iteration**: Create package.json, install dependencies (React/Vite recommended)
4. **Read the behavior spec**: Each iteration focuses on one behavior from behaviors/
5. **Implement the functionality**: Build whatever is needed to make that user behavior work
6. **Do NOT break existing behaviors** — check that your changes integrate with previously implemented features
7. **Update progress.txt** — append a brief summary of what you implemented (files changed, components added, logic wired up)
8. **Commit your changes**: Make a git commit after each behavior is complete
9. **Exit immediately**: Do NOT run the dev server - just implement and commit

**IMPORTANT**: The app will be tested separately. Do NOT run 'npm run dev' or start any servers.

## Important Notes

- The behavior specs describe USER actions (navigate, type, click) and expected outcomes
- You must implement the underlying CODE that makes those actions possible
- For example, if a behavior says "Click the delete button", you need to create a delete button and wire up the delete functionality
- Focus on making the app WORK, not on perfect code structure
- **progress.txt** and **prd.json** are your source of truth for what's been done — READ THEM
